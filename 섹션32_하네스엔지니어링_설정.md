# 섹션 32. 하네스 엔지니어링 - 설정 심화

> **이 섹션을 마치면**: settings.json과 훅 시스템을 완전히 이해하고 프로덕션 수준의 하네스를 설정할 수 있습니다.

---

## settings.json 완전 정복

### 설정 파일 계층

```
우선순위 (높음 → 낮음):

1. .claude/settings.local.json  ← 내 컴퓨터 전용 (gitignore)
2. .claude/settings.json        ← 팀 공유
3. ~/.claude/settings.json      ← 내 모든 프로젝트
```

### 완전한 settings.json 예시

```json
{
  "permissions": {
    "allow": [
      "Bash(npm test)",
      "Bash(npm run *)",
      "Bash(git status)",
      "Bash(git add *)",
      "Bash(git commit *)",
      "Bash(git push)",
      "Bash(git pull)",
      "Bash(npx prettier *)",
      "Bash(npx eslint *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(sudo *)",
      "Bash(curl * | bash *)",
      "Edit(.env)",
      "Edit(.env.local)",
      "Write(.env*)"
    ]
  },

  "env": {
    "NODE_ENV": "development",
    "NEXT_TELEMETRY_DISABLED": "1"
  },

  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "FILE=$CLAUDE_FILE_PATH; if [[ \"$FILE\" == *.ts ]] || [[ \"$FILE\" == *.tsx ]] || [[ \"$FILE\" == *.js ]]; then npx prettier --write \"$FILE\" 2>/dev/null; fi"
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"작업 완료!\" with title \"Claude Code\"' 2>/dev/null || true"
          }
        ]
      }
    ],
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "echo \"=== 세션 시작 $(date '+%Y-%m-%d %H:%M') ===\"; echo \"Git 상태:\"; git status --short 2>/dev/null | head -10"
          }
        ]
      }
    ]
  }
}
```

## 훅 시스템 완전 정복

### 훅 데이터 흐름

```
이벤트 발생
    ↓
훅 매처(matcher) 검사
    ↓
훅 실행
    ↓
결과에 따라:
  exit 0 → 계속 진행 (stdout은 Claude에게 컨텍스트로 전달)
  exit 2 → 차단 (stderr 내용이 사용자에게 표시)
  다른 exit → 에러 (무시하고 진행)
```

### 훅 타입별 상세 설명

#### 1. command 훅

```json
{
  "type": "command",
  "command": "shell 명령어",
  "timeout": 30000
}
```

#### 2. prompt 훅 (Claude가 판단)

```json
{
  "type": "prompt",
  "prompt": "이 명령어가 안전한지 판단해서 안전하면 '승인', 위험하면 '거부'라고만 답해줘"
}
```

#### 3. http 훅 (웹훅 전송)

```json
{
  "type": "http",
  "url": "https://hooks.slack.com/services/...",
  "method": "POST"
}
```

## 고급 훅 패턴

### 패턴 1: 스마트 파일 보호

```bash
#!/bin/bash
# .claude/hooks/protect-sensitive.sh

PROTECTED_PATTERNS=(".env" "secrets" "credentials" "private-key")
FILE_PATH="$CLAUDE_FILE_PATH"

for pattern in "${PROTECTED_PATTERNS[@]}"; do
  if [[ "$FILE_PATH" == *"$pattern"* ]]; then
    echo "보호된 파일입니다: $FILE_PATH" >&2
    echo "이 파일을 수정하려면 직접 편집하세요." >&2
    exit 2
  fi
done
```

```json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Edit|Write",
      "hooks": [{
        "type": "command",
        "command": "bash .claude/hooks/protect-sensitive.sh"
      }]
    }]
  }
}
```

### 패턴 2: 자동 테스트 게이트

```bash
#!/bin/bash
# .claude/hooks/test-gate.sh
# Stop 이벤트: 코드 변경 후 테스트 자동 실행

CHANGED=$(git diff --name-only 2>/dev/null | grep -E '\.(ts|tsx|js|jsx)$' | wc -l)

if [ "$CHANGED" -gt "0" ]; then
  echo "변경된 파일이 있습니다. 테스트를 실행합니다..."
  npm test --silent 2>&1
  if [ $? -ne 0 ]; then
    echo "테스트 실패! 변경사항을 확인해주세요." >&2
    exit 1
  fi
  echo "모든 테스트 통과!"
fi
```

### 패턴 3: 세션 시작 시 컨텍스트 풍부화

```bash
#!/bin/bash
# SessionStart 훅

echo "=== 프로젝트 현황 ==="
echo "현재 브랜치: $(git branch --show-current)"
echo "마지막 커밋: $(git log -1 --oneline)"
echo ""
echo "=== 오늘 변경된 파일 ==="
git diff --stat HEAD~1 2>/dev/null | head -20
echo ""
echo "=== 미완성 TODO ==="
grep -r "TODO\|FIXME\|HACK" src/ --include="*.ts" 2>/dev/null | head -10
```

## 핵심 정리

```
settings.json 핵심:
├── permissions.allow   → 자동 허용할 명령어
├── permissions.deny    → 절대 금지 명령어
├── env                 → 환경 변수
└── hooks               → 자동화 규칙

훅 마스터 패턴:
PostToolUse(Edit)  → 파일 수정 후 포맷팅
PreToolUse(Edit)   → 파일 수정 전 보안 검사
Stop               → 작업 완료 후 테스트/알림
SessionStart       → 세션 시작 시 컨텍스트 주입
```
