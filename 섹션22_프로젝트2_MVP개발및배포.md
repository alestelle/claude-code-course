# 섹션 22. [프로젝트2] MVP 개발 및 배포

> **이 섹션을 마치면**: SmartQuote의 핵심 기능을 완성하고 실제로 접근 가능한 URL에 배포합니다.

---

## MVP(최소 기능 제품)란?

```
MVP = Minimum Viable Product

"일단 동작하는 것을 빠르게 출시"
→ 피드백 받고 개선

SmartQuote MVP 범위:
✅ 견적서 폼 (고객 정보 + 품목)
✅ Notion에 자동 저장
✅ PDF 다운로드
❌ (이번 버전 제외) 이메일 발송
❌ (이번 버전 제외) 결제 연동
```

---

## MVP 개발 순서

### 1단계: 견적서 폼 완성

```
> SmartQuote 견적서 폼을 완성해줘.
  필드:
  - 고객명 (필수)
  - 이메일 (필수, 유효성 검사)
  - 회사명 (선택)
  - 품목 목록 (동적 추가/삭제 가능)
    - 항목명, 단가, 수량, 소계 자동계산
  - 부가세 (10% 자동계산)
  - 합계 (자동계산)
  - 메모 (선택)
```

### 2단계: Notion 저장 연동

```
> 폼 제출 시 Notion DB에 저장하는 기능을 만들어줘.
  Notion API를 사용하고,
  성공 시 "견적서가 저장되었습니다" 메시지 표시,
  실패 시 에러 처리해줘.
```

### 3단계: PDF 생성

```
> 저장된 견적서를 PDF로 다운로드하는 기능을 만들어줘.
  react-pdf 사용,
  회사 로고 자리, 견적서 번호, 날짜 포함.
  깔끔한 비즈니스 스타일로.
```

## Vercel 배포

### Vercel이란?

```
Vercel = Next.js 앱을 가장 쉽게 배포하는 서비스

특징:
- GitHub에 push하면 자동 배포
- 무료 플랜으로 충분
- 자동 HTTPS
- 전 세계 CDN
```

### 배포 방법

```bash
# 방법 1: Claude Code에게 요청
> Vercel에 배포해줘.
  환경변수 NOTION_API_KEY와 NOTION_DATABASE_ID를
  Vercel에 설정하는 방법도 알려줘.

# 방법 2: Vercel CLI
npm install -g vercel
vercel
```

### 환경변수 설정

```
Vercel 대시보드 → 프로젝트 → Settings → Environment Variables

추가:
NOTION_API_KEY     = [노션 API 키]
NOTION_DATABASE_ID = [노션 DB ID]
```

## 배포 후 확인

```
> 배포된 URL에서 다음을 테스트해줘:
  1. 견적서 폼 작성 → 저장 확인
  2. Notion DB에 데이터 들어갔는지 확인
  3. PDF 다운로드 테스트
  4. 모바일에서도 작동하는지 확인
```
