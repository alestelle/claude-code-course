# 섹션 30. 클로드 코드 실전 활용: 심화

> **이 섹션을 마치면**: 프로덕션 수준의 프로젝트를 Claude Code로 효율적으로 관리할 수 있습니다.

---

## 30-1. 워크트리 (Worktrees) 활용

### 워크트리란?

```
git worktree = 같은 저장소를 여러 폴더에서 동시에 작업

실무 활용:
- main 브랜치 유지 (운영 코드)
- feature 브랜치에서 개발
- 버그 수정을 핫픽스 브랜치에서

→ Claude Code로 여러 브랜치를 동시에 작업 가능
```

```bash
# 워크트리 생성
git worktree add ../eventhub-feature feature/realtime

# 각 워크트리에서 독립적으로 Claude Code 실행
cd ../eventhub-feature
claude
```

## 30-2. 대형 프로젝트 관리

### 모노레포 구조 다루기

```
eventhub/
├── apps/
│   ├── web/         ← Next.js 앱
│   └── admin/       ← 관리자 앱
├── packages/
│   ├── ui/          ← 공통 컴포넌트
│   └── api-client/  ← API 클라이언트
└── CLAUDE.md        ← 전체 구조 설명
```

```
> 모노레포 전체 구조를 파악하고
  packages/ui 의 Button 컴포넌트를
  apps/web 과 apps/admin 모두에서 사용하도록 설정해줘.
```

---

## 30-3. CI/CD 자동화

```
> GitHub Actions를 설정해줘:

  트리거: main 브랜치 push
  작업:
  1. npm test 실행
  2. 테스트 통과 시 Vercel 자동 배포
  3. 실패 시 Slack 알림

  .github/workflows/deploy.yml 파일로 만들어줘.
```

---

## 30-4. 성능 최적화

```
> EventHub 앱의 성능을 분석하고 최적화해줘:

  확인 항목:
  1. Lighthouse 점수 (목표: 90점 이상)
  2. 번들 크기 분석 (npm run analyze)
  3. 이미지 최적화
  4. 불필요한 리렌더링

  각 문제를 발견하면 즉시 수정해줘.
```

## 30-5. 실전 1인 개발자 루틴

### 아침 루틴 스킬

```markdown
# .claude/skills/morning.md
---
description: 하루 시작 시 프로젝트 상태를 파악합니다
---

다음을 순서대로 실행합니다:
1. git pull → 최신 코드 확인
2. ROADMAP.md → 오늘 해야 할 작업 파악
3. 미완성 테스트 확인
4. 오늘의 작업 계획 3가지 제안
```

실행:
```
> /morning
```

### 저녁 루틴 스킬

```markdown
# .claude/skills/evening.md
---
description: 하루 작업 마무리를 수행합니다
---

1. 오늘 변경된 파일들 확인
2. 커밋 안 된 작업 정리 및 커밋
3. ROADMAP.md 진행 상황 업데이트
4. 내일 해야 할 작업 메모
5. git push
```

## 핵심 정리

```
심화 실전 패턴:

워크트리      → 여러 브랜치 동시 작업
모노레포      → 대형 프로젝트 구조화
CI/CD         → 자동 테스트 + 배포
성능 최적화   → Lighthouse + 번들 분석
일상 루틴화  → /morning, /evening 스킬
```
