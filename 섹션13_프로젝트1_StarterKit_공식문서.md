# 섹션 13. [프로젝트1] Starter Kit 만들기 - 공식문서 활용

> **이 섹션을 마치면**: 공식 문서와 AI를 함께 활용하여 최신 방법으로 기능을 구현할 수 있습니다.

---

## 왜 공식문서가 중요한가?

```
문제:
AI는 학습 데이터 기준 시점이 있습니다.
최신 라이브러리 변경사항을 모를 수 있습니다.

해결:
공식문서를 직접 Claude에게 제공하면
가장 최신 방법으로 구현해줍니다.
```

---

## 공식문서 활용 방법 3가지

### 방법 1: URL 직접 참조 (웹 검색 MCP 필요)

```
> Next.js 14 App Router의 공식 문서를 참조해서
  서버 컴포넌트와 클라이언트 컴포넌트를 올바르게 구분해줘
```

### 방법 2: 문서 내용 붙여넣기

```
[공식 문서 일부 복사 후]

> 이 공식 문서를 참조해서 인증 기능을 구현해줘:
  [붙여넣기한 공식 문서 내용]
```

### 방법 3: @참조 기능 활용

```
# 공식 문서를 파일로 저장 후
> @docs/nextauth.md 이 문서를 참조해서
  NextAuth.js 설정해줘
```

---

## 주요 공식문서 활용 시나리오

### Next.js 14 App Router 설정

```
> Next.js 14 공식 문서의 App Router 방식으로
  검색 페이지를 서버 사이드 렌더링으로 만들어줘.
  검색어는 URL 쿼리 파라미터로 받아.
  (예: /search?q=손흥민)
```

### Tailwind CSS 반응형

```
> Tailwind CSS 공식 문서의 반응형 브레이크포인트를 사용해서
  모바일(sm), 태블릿(md), 데스크탑(lg) 각각 다른 레이아웃으로 만들어줘
```

### Prisma 스키마 설계

```
> Prisma 공식 문서를 참고해서 다음 테이블을 설계해줘:
  - 선수(Player): 이름, 포지션, 팀, 국적, 생년월일
  - 팀(Team): 이름, 리그, 국가, 창단연도
  - 경기(Match): 홈팀, 원정팀, 일시, 결과, 리그
  관계도 함께 설정해줘
```

## 실습 미션

```
1. Next.js 공식 사이트(nextjs.org)에서
   "Data Fetching" 문서 찾아서 복사

2. Claude Code에게:
   > 이 공식 문서를 참고해서 선수 목록을 가져오는
     서버 컴포넌트를 만들어줘:
     [붙여넣기]

3. 결과 확인 후 /simplify 로 코드 품질 검토
```
