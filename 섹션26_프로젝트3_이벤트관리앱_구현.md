# 섹션 26. [프로젝트3] 이벤트 관리 앱 - 구현하기

> **이 섹션을 마치면**: EventHub의 전체 MVP를 완성하고 배포까지 완료합니다.

---

## 전체 구현 흐름

```
1. Supabase 테이블 생성
2. 인증 구현 (주최자 로그인)
3. 이벤트 생성 페이지
4. 참가 신청 페이지 (공개)
5. 주최자 대시보드
6. 실시간 참가 현황
7. 배포
```

---

## 1단계: Supabase 테이블 생성

```
> Supabase SQL Editor에서 실행할 SQL을 작성해줘:

  테이블:
  1. events
     - id, title, description, date, location
     - max_participants, organizer_id (FK)
     - created_at, is_active

  2. registrations
     - id, event_id (FK), name, email, phone
     - registered_at, checked_in_at (null)

  RLS 정책:
  - events: 주최자만 생성/수정, 모두 조회 가능
  - registrations: 이벤트별 주최자만 조회, 누구나 등록
```

---

## 2단계: 이벤트 생성 기능

```
> /new-page event-create:

  /events/new 페이지를 만들어줘:
  - 로그인한 사용자만 접근
  - 이벤트 생성 폼
    (제목, 날짜/시간, 장소, 설명, 최대 인원, 이미지)
  - 저장 시 Supabase에 저장
  - 저장 후 /events/[id] 로 이동
```

---

## 3단계: 참가 신청 페이지 (공개)

```
> /events/[id] 공개 페이지를 만들어줘:
  - 이벤트 상세 정보
  - 참가 신청 폼 (이름, 이메일, 전화번호)
  - 현재 신청 인원 / 최대 인원 표시
  - 마감 시 "신청이 마감되었습니다" 표시
  - 신청 완료 후 확인 화면
```

---

## 4단계: 주최자 대시보드

```
> /dashboard 페이지를 만들어줘 (로그인 필요):

  기능:
  - 내 이벤트 목록
  - 이벤트별 참가자 수 / 정원 바 표시
  - 이벤트 클릭 → 참가자 목록

  /dashboard/events/[id] 참가자 관리:
  - 참가자 목록 (이름, 이메일, 신청일)
  - 입장 체크인 버튼
  - CSV 다운로드
  - Supabase Realtime으로 실시간 업데이트
```

---

## 5단계: 실시간 참가 현황

```
> 주최자 대시보드에 실시간 기능을 추가해줘:
  Supabase Realtime을 사용해서
  새 참가자가 등록되면 자동으로 화면 업데이트.
  신규 등록 시 토스트 알림 표시.
```

---

## 6단계: 배포

```
> EventHub를 Vercel에 배포해줘.
  필요한 환경변수 목록과 설정 방법 알려줘.
  배포 후 실제 테스트:
  1. 이벤트 생성
  2. 참가 링크로 신청
  3. 대시보드에서 실시간 확인
```
