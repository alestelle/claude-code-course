# 섹션 28. 주제별 심화

> **이 섹션을 마치면**: 실무에서 자주 마주치는 고급 패턴들을 처리할 수 있습니다.

---

## 28-1. 대용량 코드베이스 다루기

### 컨텍스트 관리 전략

```
문제: 프로젝트가 커지면 Claude Code가 전체를 한 번에 처리 못함

해결 전략:

1. CLAUDE.md에 폴더 구조 명시
   "핵심 파일: src/lib/db.ts, src/api/routes.ts"

2. 서브에이전트로 분할 처리
   > 인증 모듈만 서브에이전트로 분석해줘

3. 스킬로 모듈별 작업
   > /update-auth  (인증 관련 작업만)
   > /update-ui    (UI 관련 작업만)

4. /compact 적극 활용
   대화가 길어지면 주기적으로 압축
```

---

## 28-2. 반복 작업 자동화

### /loop 활용

```
> /loop 30m /check
→ 30분마다 코드 품질 검사 자동 실행
```

### /schedule 활용

```
> /schedule
  매일 오전 9시에 다음을 실행해줘:
  - git pull로 최신 코드 받기
  - npm test 실행
  - 테스트 실패 시 Slack 알림
```

---

## 28-3. 효과적인 프롬프트 작성

### 기획자 관점의 프롬프트 원칙

```
❌ 나쁜 프롬프트:
"검색 기능 고쳐줘"

✅ 좋은 프롬프트:
"검색 페이지에서 한글 초성 검색이 안 되는 버그를 고쳐줘.
예: '손흥' 입력 시 '손흥민' 결과가 나와야 하는데 안 나옴.
src/api/search/route.ts 의 쿼리 로직을 먼저 확인해줘.
수정 전에 어떻게 고칠지 계획을 먼저 알려줘."
```

### 검증 기준 포함하기

```
> 로그인 기능을 구현해줘.
  완료 기준:
  - 이메일/비밀번호로 로그인 가능
  - 잘못된 비밀번호 시 에러 메시지
  - 로그인 상태 유지 (새로고침 후에도)
  - Cypress 테스트 코드도 작성해줘
```

---

## 28-4. 오류 처리 패턴

```
오류가 발생했을 때:

1. 에러 메시지 그대로 붙여넣기
   > 이 에러가 발생했어: [에러 전체 내용]
     어디서 왜 발생하는지 분석하고 고쳐줘

2. 단계적 접근
   > 일단 에러 원인부터 파악해줘.
     수정하기 전에 설명 먼저.

3. 재현 방법 제공
   > /api/search?q=손흥민 호출 시 500 에러 발생.
     curl 명령어로 재현 가능해.
```

---

## 28-5. 테스트 주도 개발 (TDD)

```
> 검색 API를 TDD 방식으로 만들어줘:
  1. 실패하는 테스트 먼저 작성
  2. 테스트를 통과하는 최소 코드 작성
  3. 리팩토링

  테스트 프레임워크: Jest
  커버리지 목표: 80% 이상
```

---

## 핵심 정리

```
실무 심화 패턴:

대용량 코드베이스 → 서브에이전트 분할 + 스킬화
반복 작업 → /loop, /schedule 자동화
좋은 프롬프트 → 구체적 + 검증 기준 포함
오류 처리 → 에러 전체 붙여넣기 + 단계적 접근
```
