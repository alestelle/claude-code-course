# 섹션 9. 설정 파일과 메모리 관리

> **이 섹션을 마치면**: Claude Code가 매 세션마다 프로젝트를 '기억'하도록 설정할 수 있습니다.

---

## 9-1. Claude Code의 메모리 문제

### 문제: Claude는 대화를 기억 못합니다

```
오늘 세션:
나: "이 프로젝트는 스포츠 검색 서비스야. React로 만들고 있어."
Claude: "알겠습니다!"

내일 새 세션:
나: "버튼 색상 바꿔줘"
Claude: "어떤 프로젝트인가요? 어떤 기술을 사용하시나요?"
```

매번 프로젝트 설명을 반복해야 하는 문제가 생깁니다.

### 해결책: CLAUDE.md

```
CLAUDE.md = Claude Code에게 주는 '영구 메모'

이 파일을 만들어두면, 매 세션마다 자동으로 읽어서
프로젝트 맥락을 항상 기억합니다.
```

---

## 9-2. CLAUDE.md 만들기

### 자동 생성 (추천)

```
> /init
```

Claude Code가 현재 프로젝트를 분석해서 자동으로 CLAUDE.md를 만들어줍니다.

### 수동 작성 예시

```markdown
# 프로젝트: 스포츠 검색 서비스

## 프로젝트 개요
선수, 팀, 경기 정보를 검색할 수 있는 웹 서비스.
IT 기획자가 Claude Code와 함께 개발 중입니다.

## 기술 스택
- Frontend: React 18, TypeScript
- Backend: Node.js, Express
- Database: PostgreSQL
- 배포: Vercel (프론트), Railway (백엔드)

## 주요 명령어
- 개발 서버: `npm run dev`
- 빌드: `npm run build`
- 테스트: `npm test`

## 코딩 규칙
- 함수명은 camelCase
- 컴포넌트명은 PascalCase
- 주석은 한국어로 작성
- 커밋 메시지 형식: feat/fix/docs/refactor: 설명

## 현재 작업 중인 기능
- [ ] 선수 검색 자동완성
- [ ] 경기 결과 실시간 업데이트
```

## 9-3. 설정 파일 구조

Claude Code는 여러 단계의 설정 파일을 사용합니다:

```
우선순위 (높음 → 낮음):

.claude/settings.local.json  ← 내 컴퓨터에서만 적용 (Git 제외)
.claude/settings.json        ← 프로젝트 팀 공유 설정
~/.claude/settings.json      ← 내 모든 프로젝트에 적용
```

### 기본 settings.json 구성

```json
{
  "permissions": {
    "allow": ["Bash(npm *)", "Bash(git *)"],
    "deny": ["Bash(rm -rf *)", "Edit(.env)"]
  },
  "env": {
    "NODE_ENV": "development"
  }
}
```

## 9-4. 자동 메모리 (Auto Memory)

Claude Code는 대화 중에 중요한 정보를 자동으로 메모리 파일에 저장할 수 있습니다.

### 메모리 파일 위치

```
~/.claude/projects/[프로젝트경로]/memory/
  ├── user.md       ← 사용자 선호도, 작업 방식
  ├── project.md    ← 프로젝트 관련 중요 사항
  └── feedback.md   ← 잘된 것, 수정이 필요한 것
```

### 메모리 확인 및 편집

```
> /memory
```

저장된 모든 메모리 파일을 확인하고 편집할 수 있습니다.

### 직접 메모리 저장 요청

```
> 기억해줘: 이 프로젝트에서 CSS는 항상 Tailwind로 작성해야 해

> 기억해줘: 데이터베이스는 Supabase를 사용하고
  테이블명은 스네이크 케이스(snake_case)로 통일해

> 앞으로는 절대로 console.log를 코드에 남기지 마
```

## 9-5. CLAUDE.md 파일 계층 구조

여러 폴더에 CLAUDE.md를 만들 수 있습니다:

```
my-project/
├── CLAUDE.md                 ← 전체 프로젝트 설정
├── src/
│   ├── CLAUDE.md             ← src 폴더 전용 규칙
│   └── components/
│       └── CLAUDE.md         ← 컴포넌트 전용 규칙
└── api/
    └── CLAUDE.md             ← API 전용 규칙
```

**예시: src/components/CLAUDE.md**
```markdown
# 컴포넌트 작성 규칙

- 모든 컴포넌트는 TypeScript로 작성
- Props 타입은 반드시 명시
- 스타일은 Tailwind CSS 클래스만 사용
- 각 컴포넌트는 단일 책임 원칙 준수
```

## 9-6. 실전 CLAUDE.md 작성 가이드

### 꼭 포함할 내용

```markdown
# [프로젝트명]

## 프로젝트 개요
무엇을 위한 서비스인지 (2-3문장)

## 기술 스택
사용하는 모든 기술

## 주요 명령어
자주 실행하는 npm 스크립트

## 코딩 규칙
팀 또는 개인 코딩 컨벤션

## 폴더 구조
주요 폴더와 역할 설명

## 현재 작업 상황
지금 개발 중인 기능 (선택)
```

### 넣지 않아도 되는 내용

```
❌ 너무 긴 설명 (200줄 이하 권장)
❌ 코드 예시 (스킬 파일로 분리)
❌ 자주 바뀌는 임시 정보
```

## 9-7. 실습 미션

```
> /init 실행

생성된 CLAUDE.md를 열어서 다음을 추가해줘:
1. 이 프로젝트는 스포츠 검색 서비스야
2. 모든 주석은 한국어로 달아줘
3. npm run dev로 개발 서버 실행해
4. 커밋 메시지는 한국어로 작성해줘

수정 후 /memory 로 내용 확인해줘
```

---

## 핵심 정리

```
CLAUDE.md      = 프로젝트 영구 메모 (매 세션마다 자동 읽음)
settings.json  = 권한, 환경변수 등 동작 설정
자동 메모리    = 대화 중 중요 사항 자동 저장

/init   → CLAUDE.md 자동 생성
/memory → 현재 저장된 메모리 확인
```
