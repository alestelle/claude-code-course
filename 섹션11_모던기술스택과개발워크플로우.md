# 섹션 11. 모던 기술스택과 개발 워크플로우

> **이 섹션을 마치면**: AI와 협업할 때 알아야 할 기술 지형을 이해하고, 효율적인 개발 흐름을 만들 수 있습니다.

---

## 11-1. 기획자가 알아야 할 기술 지형

### 웹 서비스의 3계층

```
[사용자 브라우저]
      ↕  (HTTP 요청/응답)
[프론트엔드] ← HTML/CSS/JavaScript, React, Vue
      ↕  (API 호출)
[백엔드]     ← Node.js, Python, Java
      ↕  (쿼리)
[데이터베이스] ← PostgreSQL, MySQL, MongoDB
```

### 기술 스택 선택 가이드

| 상황 | 추천 스택 | 이유 |
|------|----------|------|
| 빠른 프로토타입 | Next.js + Supabase | 풀스택, 배포 쉬움 |
| 대용량 검색 서비스 | React + Elasticsearch | 검색 최적화 |
| 관리자 도구 | Next.js + PostgreSQL | 안정성 |
| 모바일 앱 | React Native | 코드 재사용 |

## 11-2. 스포츠 검색 서비스 기술 스택 예시

```
프론트엔드:
  Next.js 14     ← React 기반 프레임워크, SEO 최적화
  TypeScript     ← 타입 안전성

  PostgreSQL     ← 관계형 DB, 검색 기능 강력
  Redis          ← 빠른 캐싱 (검색 성능)

배포:
  Vercel         ← 프론트엔드 호스팅 (자동 배포)
  Railway        ← 백엔드/DB 호스팅
```

## 11-3. AI 시대의 개발 워크플로우

### 전통적 워크플로우 vs AI 워크플로우

```
전통적 방식:
기획 → 디자인 → 개발 → 테스트 → 배포
(각 단계 전환에 많은 시간 소요)

AI 워크플로우:
기획 ↔ AI 프로토타입 ↔ 테스트 ↔ 개선 ↔ 배포
(사이클이 훨씬 빠름, 기획자도 직접 프로토타입 가능)
```

---

## 11-4. Claude Code 기반 개발 워크플로우

### 기획자 + AI 개발 1인 프로세스

```
1. 아이디어 → PRD 작성 (/prd 스킬 활용)

2. Claude Code로 초안 생성
   > 이 PRD를 보고 프로젝트 뼈대를 만들어줘

3. 피드백 반영
   > 검색 결과 화면이 너무 복잡해. 더 단순하게 만들어줘

4. 기능 추가
   > 필터 기능 추가해줘

5. 배포
   > Vercel에 배포해줘

전체 사이클: 아이디어 → 서비스 = 수 시간
```

## 11-5. 효율적인 Claude Code 사용 패턴

### 패턴 1: 탐색 먼저, 수정 나중

```
❌ 나쁜 방법:
> 로그인 버그 고쳐줘 (무엇을 고쳐야 할지 모르는 상태)

✅ 좋은 방법:
> 로그인 관련 파일들을 먼저 분석해줘
  어디서 문제가 생기는지 파악하고 나서 수정 계획 알려줘
  (계획 확인 후)
> 그 방법으로 수정해줘
```

### 패턴 2: 검증 기준 먼저 제시

```
✅ 좋은 방법:
> 검색 자동완성 기능을 만들어줘.
  성공 기준:
  - 3글자 이상 입력 시 자동완성 목록 표시
  - 목록 클릭 시 검색 실행
  - 500ms 이내 응답
  - 테스트 코드도 같이 작성해줘
```

### 패턴 3: 컨텍스트 효율 관리

```
대화가 길어질 때:
1. /compact → 대화 압축
2. /context → 사용량 확인
3. 핵심 정보는 CLAUDE.md에 기록
```

## 11-6. 기획자가 Claude Code로 할 수 있는 것들

```
✅ 가능한 것:
- HTML/CSS 랜딩 페이지 제작
- 간단한 관리자 대시보드
- API 연동 (데이터 가져와서 표시)
- 데이터베이스 설계 및 구축
- 자동화 스크립트 작성

⚠️ 어려운 것 (가능하지만 시간 필요):
- 복잡한 실시간 기능
- 대규모 트래픽 최적화
- 복잡한 알고리즘

❌ Claude Code가 대신 못하는 것:
- 서비스 방향 결정
- 사용자 리서치
- 비즈니스 전략
→ 이건 여전히 기획자의 영역!
```

## 핵심 정리

```
기술 스택 = 서비스를 만드는 도구들의 조합

```
