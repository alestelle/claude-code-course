# 섹션 33. 하네스 엔지니어링 - 실습

> **이 섹션을 마치면**: 실무용 완전 자동화 하네스를 처음부터 구축할 수 있습니다.

---

## 최종 실습 목표

```
이 섹션에서 만들 것:

"스포츠 검색 서비스를 위한 프로덕션 수준 하네스"

포함 요소:
✅ CLAUDE.md (프로젝트 메모리)
✅ settings.json (권한 + 자동화 훅)
✅ 스킬 라이브러리 (10개 커맨드)
✅ 자동화 훅 (5개)
✅ MCP 연결 (brave-search + github)
✅ 팀 공유 설정
```

---

## 실습 1: 기반 설정 구축

### 1-1. CLAUDE.md 완성

```
> /init 실행 후 CLAUDE.md를 다음 내용으로 완성해줘:

  프로젝트: 스포츠 검색 서비스
  스택: Next.js 14, TypeScript, Tailwind, Supabase
  주 명령어: npm run dev / npm test / npm run build
  코딩 규칙:
  - TypeScript strict 모드
  - 함수형 컴포넌트만 사용
  - 커밋 메시지: feat/fix/docs/refactor: 한국어 설명
  - 모든 API는 에러 처리 필수
  핵심 파일:
  - src/lib/supabase.ts (DB 연결)
  - src/app/api/ (API 라우트)
  - src/components/ (공통 컴포넌트)
```

### 1-2. settings.json 완성

```
> .claude/settings.json을 만들어줘:
  허용: npm 명령어 전체, git 기본 명령어
  차단: rm -rf, sudo, .env 편집
  훅:
    - 파일 수정 시 prettier 자동 실행
    - 작업 완료 시 맥 알림
    - 세션 시작 시 git status + 오늘 날짜 출력
```

---

## 실습 2: 스킬 라이브러리 구축

```
> 다음 10개 스킬을 .claude/skills/ 에 만들어줘:

개발 스킬:
1. /new-page [이름]    → 표준 페이지 생성
2. /new-component [이름] → 표준 컴포넌트 생성
3. /add-api [이름]     → API 엔드포인트 생성
4. /add-table [이름]   → Supabase 테이블 + RLS

리뷰 스킬:
5. /review             → 변경 파일 종합 리뷰
6. /security           → 보안 취약점 체크

작업 관리 스킬:
7. /prd [아이디어]     → PRD 자동 작성
8. /morning            → 하루 시작 루틴
9. /evening            → 하루 마무리 루틴

도움말:
10. /help              → 전체 명령어 목록
```

---

## 실습 3: 고급 훅 설정

```
> .claude/hooks/ 폴더를 만들고 다음 훅을 설정해줘:

1. protect-env.sh
   .env, .env.local 파일 수정 시도 차단

2. auto-format.sh
   TS/JS 파일 수정 시 prettier + eslint 자동 실행

3. test-on-stop.sh
   작업 완료 후 변경 파일 있으면 관련 테스트 실행

4. session-briefing.sh
   세션 시작 시:
   - 현재 브랜치
   - 미완성 TODO 수
   - 마지막 커밋 시간
   - 오늘 날짜와 인사

settings.json에 모두 등록해줘.
```

---

## 실습 4: MCP 연동

```
> .claude/mcp.json을 만들어줘:
  1. brave-search (환경변수로 API 키 관리)
  2. github (환경변수로 토큰 관리)

  연결 후 테스트:
  - "K리그 이번 주 경기 결과" 검색
  - GitHub에 "하네스 설정 완료" 이슈 생성
```

## 실습 5: 팀 공유 하네스 패키지

### 최종 폴더 구조

```
.claude/
├── settings.json          ← 팀 공유 (git에 포함)
├── settings.local.json    ← 개인 설정 (.gitignore)
├── mcp.json               ← MCP 설정 (git에 포함, 키 제외)
├── hooks/
│   ├── protect-env.sh
│   ├── auto-format.sh
│   ├── test-on-stop.sh
│   └── session-briefing.sh
└── skills/
    ├── new-page.md
    ├── new-component.md
    ├── add-api.md
    ├── add-table.md
    ├── review.md
    ├── security.md
    ├── prd.md
    ├── morning.md
    ├── evening.md
    └── help.md
```

```
> 이 구조를 완성하고
  .gitignore에 settings.local.json과 .env*를 추가해줘.
  README에 팀원을 위한 하네스 사용 가이드도 작성해줘.
```

## 실습 6: 하네스 검증

### 동작 테스트

```bash
# 1. 세션 시작 훅 확인
claude
# → 세션 브리핑이 출력되어야 함

# 2. 스킬 테스트
> /help
# → 전체 스킬 목록 출력

# 3. 파일 보호 테스트
> .env 파일을 수정해줘
# → "보호된 파일입니다" 메시지 + 차단

# 4. 자동 포맷팅 테스트
> test.ts 파일을 만들고 일부러 포맷 안 맞게 작성해줘
# → 저장 후 자동으로 prettier 실행됨

# 5. 알림 테스트
> /morning 실행해줘
# → 작업 완료 시 맥 알림
```

## 완성된 하네스 치트시트

```
하네스 파일 위치:
.claude/settings.json     ← 핵심 설정
.claude/mcp.json          ← 외부 연결
.claude/hooks/            ← 자동화 스크립트
.claude/skills/           ← 커스텀 명령어
CLAUDE.md                 ← 프로젝트 메모리

주요 명령어:
/help      → 스킬 목록
/morning   → 하루 시작
/evening   → 하루 마무리
/review    → 코드 리뷰
/prd       → PRD 작성

자동으로 일어나는 것:
파일 수정 → prettier 자동 실행
작업 완료 → 알림 + 테스트
세션 시작 → 컨텍스트 브리핑
.env 수정 → 자동 차단
```
