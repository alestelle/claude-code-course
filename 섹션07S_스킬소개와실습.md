# 섹션 7-S. 클로드 스킬(Skill) 소개 및 실습 ⭐

> **이 섹션을 마치면**: 나만의 슬래시 명령어(스킬)를 만들고, 반복 작업을 자동화할 수 있습니다.

---

## 스킬이란 무엇인가?

### 한 줄 정의

> **스킬 = 내가 직접 만드는 커스텀 슬래시 명령어**

```
기본 제공 명령어:  /init, /model, /clear ...
내가 만든 명령어:  /prd, /review, /deploy ...
```

### 기획자 관점의 비유

```
스킬 = 업무 매뉴얼의 자동화 버전

예를 들어:
신규 기획서를 쓸 때마다 같은 양식을 쓴다면
→ /prd 명령어 하나로 양식 자동 생성

코드 리뷰를 매번 같은 기준으로 한다면
→ /review 명령어 하나로 체크리스트 자동 실행

배포할 때마다 같은 절차를 따른다면
→ /deploy 명령어 하나로 전체 절차 자동 실행
```

---

## 스킬의 구조

스킬은 `.claude/skills/` 폴더 안에 **마크다운 파일**로 만듭니다.

```
my-project/
└── .claude/
    └── skills/
        ├── prd.md          ← /prd 명령어
        ├── review.md       ← /review 명령어
        └── deploy.md       ← /deploy 명령어
```

### 스킬 파일 기본 형식

```markdown
---
description: 이 스킬이 무엇을 하는지 한 줄 설명
---

# 스킬 제목

여기에 Claude에게 줄 지시사항을 작성합니다.
마크다운 형식으로 자유롭게 작성 가능합니다.

## 수행할 작업

1. 첫 번째 단계
2. 두 번째 단계
3. 세 번째 단계
```

---

## 실습 1: 첫 번째 스킬 만들기

### PRD(제품 요구사항 문서) 자동 생성 스킬

기획자에게 친숙한 PRD 작성을 자동화해봅시다.

```bash
# 스킬 폴더 만들기
mkdir -p .claude/skills

# Claude Code에서 스킬 파일 만들기
claude
> .claude/skills/prd.md 파일을 만들어줘.
  이 스킬은 /prd 라는 이름으로 실행되며,
  사용자가 기능 아이디어를 말하면
  다음 형식으로 PRD 문서를 작성해줘:
  - 개요
  - 목표 사용자
  - 핵심 기능 (우선순위별)
  - 성공 지표 (KPI)
  - 제외 범위 (Out of Scope)
  IT 기획자가 바로 사용할 수 있는 수준으로 작성
```

### 만들어진 스킬 파일 예시

```markdown
---
description: 기능 아이디어를 받아 PRD 문서를 자동 작성합니다
---

# PRD 자동 작성 스킬

사용자가 설명하는 기능 아이디어를 바탕으로
전문적인 PRD(Product Requirements Document)를 작성합니다.

## 출력 형식

### 1. 개요
[기능의 목적과 배경을 2-3문장으로 설명]

### 2. 목표 사용자
[주요 사용자 페르소나 2-3가지]

### 3. 핵심 기능
| 우선순위 | 기능 | 설명 |
|---------|------|------|
| P0 (필수) | ... | ... |
| P1 (중요) | ... | ... |
| P2 (추후) | ... | ... |

### 4. 성공 지표 (KPI)
- [ ] 지표 1: ...
- [ ] 지표 2: ...

### 5. 제외 범위
- 이번 버전에서 다루지 않는 것들

## 주의사항
- 기술적 구현 방법보다 사용자 가치에 집중
- 모든 기능은 측정 가능한 성공 지표와 연결
```

### 스킬 실행하기

```
> /prd 스포츠 경기 결과 알림 서비스
```

## 실습 2: 코드 리뷰 스킬

```markdown
# .claude/skills/review.md

---
description: 코드 품질을 검토하고 개선사항을 제안합니다
---

# 코드 리뷰 스킬

현재 프로젝트의 코드를 검토하고 다음 기준으로 평가합니다:

## 검토 항목

1. **보안**: API 키 노출, SQL 인젝션, XSS 취약점
2. **성능**: 불필요한 반복, 메모리 누수, 느린 쿼리
3. **가독성**: 변수명, 함수 길이, 주석 적절성
4. **유지보수성**: 중복 코드, 테스트 커버리지

## 출력 형식

각 항목을 🔴(심각) 🟡(주의) 🟢(양호)로 표시하고
구체적인 개선 방법을 제안합니다.
```

실행:
```
> /review src/api/auth.js 파일을 검토해줘
```

## 실습 3: 기획자 전용 스킬 세트

### 스포츠 검색 서비스 기획 전용 스킬

```markdown
# .claude/skills/sports-spec.md

---
description: 스포츠 검색 서비스 기능 명세서를 작성합니다
---

# 스포츠 검색 기능 명세 스킬

이 프로젝트는 스포츠 관련 검색 서비스입니다.
기능 명세서를 작성할 때 다음을 반영합니다:

## 서비스 컨텍스트
- 주요 콘텐츠: 선수 정보, 팀 정보, 경기 결과, 순위
- 타겟 사용자: 스포츠 팬, 스포츠 기자, 통계 분석가
- 핵심 가치: 정확성, 속도, 포괄성

## 명세서 형식
[기능명], [사용자 스토리], [수용 기준], [데이터 요구사항]
순서로 작성합니다.
```

## 전역 스킬 vs 프로젝트 스킬

```
전역 스킬 (모든 프로젝트에서 사용):
~/.claude/skills/

프로젝트 스킬 (해당 프로젝트에서만 사용):
.claude/skills/
```

| 구분 | 위치 | 언제 사용 |
|------|------|----------|
| 전역 스킬 | `~/.claude/skills/` | 어디서든 쓰는 공통 작업 |
| 프로젝트 스킬 | `.claude/skills/` | 이 프로젝트 전용 작업 |

## 스킬 활용 팁

### 팁 1: 파일 참조(@) 활용

```markdown
# .claude/skills/analyze.md
---
description: 프로젝트 구조를 분석합니다
---

@package.json 과 @README.md 를 참조하여
현재 프로젝트의 기술 스택과 목적을 분석하고
개선이 필요한 부분을 제안하세요.
```

### 팁 2: 서브에이전트로 실행

긴 작업은 서브에이전트에서 격리 실행:

```markdown
---
description: 전체 코드베이스 리팩토링
run_in_subagent: true
---
```

### 팁 3: 스킬 체이닝

```
> /prd 알림 서비스 → PRD 작성 완료
> /review          → 코드 리뷰 실행
> /deploy          → 배포 실행
```

---

## 핵심 정리

```
스킬 = 내가 만든 /명령어

만드는 방법:
1. .claude/skills/ 폴더에 .md 파일 생성
2. --- 사이에 description 작성
3. 본문에 Claude에게 줄 지시사항 작성
4. /스킬이름 으로 실행

활용 아이디어 (기획자용):
/prd      → 제품 요구사항 문서 자동 작성
/review   → 코드 리뷰 자동화
/standup  → 데일리 스탠드업 요약
/spec     → 기능 명세서 작성
/qa       → QA 체크리스트 생성
```
