# 섹션 6. 클로드 코드 권한

> **이 섹션을 마치면**: Claude Code가 무엇을 할 수 있고 없는지 직접 제어할 수 있습니다.

---

## 6-1. 권한 시스템이란?

Claude Code는 강력합니다. 파일을 만들고, 지우고, 명령을 실행할 수 있습니다. 따라서 **"무엇을 허용할 것인가"** 를 내가 통제해야 합니다.

### 비유로 이해하기

```
Claude Code = 능력 있는 신입 직원

- 지시한 것만 하게 할 수도 있고 (엄격 모드)
- 자율적으로 처리하게 할 수도 있고 (자동 모드)
- 중요한 것만 물어보게 할 수도 있습니다 (기본 모드)

처음에는 기본 모드로 시작하고,
익숙해지면 점점 자율권을 늘려가세요.
```

---

## 6-2. 권한 모드 4가지

### Shift + Tab으로 모드 전환

터미널에서 Claude Code 실행 중에 `Shift + Tab`을 누르면 모드가 순환됩니다.

```
기본 모드 → 편집 허용 모드 → 계획 모드 → 자동 모드
```

| 모드 | 특징 | 언제 사용 |
|------|------|----------|
| **기본 (Default)** | 파일 수정·명령 실행 전마다 허가 요청 | 처음 배울 때, 중요한 작업 |
| **편집 허용** | 파일 수정은 자동, 명령 실행만 물어봄 | 코딩 작업이 많을 때 |
| **계획 모드** | 읽기만 하고 수정 안 함, 계획만 보여줌 | 큰 작업 시작 전 검토 |
| **자동 모드** | 거의 모든 것을 자동 처리 | 믿을 수 있는 환경에서 |

---

## 6-3. 허가 요청 읽는 법

Claude Code가 무언가를 하려 할 때 이런 화면이 나타납니다:

```
╔══════════════════════════════════════╗
║  Claude wants to run a command:      ║
║                                      ║
║  npm run build                       ║
║                                      ║
║  ❯ Allow once                        ║
║    Allow always                      ║
║    Deny                              ║
╚══════════════════════════════════════╝
```

| 선택 | 의미 | 언제 선택 |
|------|------|----------|
| Allow once | 이번 한 번만 허용 | 처음 보는 명령 |
| Allow always | 앞으로도 항상 허용 | 자주 쓰는 안전한 명령 |
| Deny | 거부 | 의도하지 않은 명령 |

---

## 6-4. 권한 규칙 설정하기

### 허용 목록 (Allow List)

자주 쓰는 안전한 명령을 미리 허용합니다.

`.claude/settings.json` 파일에 설정:

```json
{
  "permissions": {
    "allow": [
      "Bash(npm test)",
      "Bash(npm run dev)",
      "Bash(npm run build)",
      "Bash(git status)",
      "Bash(git add *)",
      "Bash(git commit *)"
    ]
  }
}
```

### 차단 목록 (Deny List)

절대 실행하면 안 되는 위험한 명령을 미리 차단합니다:

```json
{
  "permissions": {
    "deny": [
      "Bash(rm -rf *)",
      "Bash(sudo *)",
      "Bash(curl * | bash *)"
    ]
  }
}
```

---

## 6-5. 파일 보호하기

중요한 파일은 Claude Code가 수정하지 못하게 막을 수 있습니다:

```json
{
  "permissions": {
    "deny": [
      "Edit(.env)",
      "Edit(.env.local)",
      "Write(.env*)"
    ]
  }
}
```

### 반드시 보호해야 할 파일들

```
.env              ← API 키, 비밀번호 등 민감 정보
.env.local        ← 로컬 환경 변수
config/secrets.*  ← 각종 시크릿 파일
```

---

## 6-6. 권한 모드 실전 활용법

### 상황별 권장 모드

**새 프로젝트 시작할 때**
```
계획 모드 → 먼저 Claude의 계획 확인 → 기본 모드로 실행
```

**믿을 수 있는 반복 작업**
```
편집 허용 모드 → 파일 수정은 자동, 명령만 확인
```

**큰 변경사항 적용 전**
```
계획 모드로 먼저 확인:
> /plan 로그인 시스템을 OAuth2로 교체해줘
(실제 변경 없이 계획만 보여줌)
```

---

## 6-7. 체크포인트 활용 (실수 복구)

Claude Code는 변경 전 자동으로 체크포인트를 만듭니다.

```
실수로 잘못된 변경이 됐다면?
→ Esc + Esc 를 두 번 눌러 이전 상태로 복원
```

```
> Esc  (첫 번째)
  되돌릴까요? 마지막 변경사항을 취소합니다.

> Esc  (두 번째)
  ✓ 이전 상태로 복원됨
```

---

## 6-8. 실습 미션

### 미션: 권한 설정 파일 만들기

```bash
# 1. 프로젝트 폴더에서 설정 폴더 생성
mkdir -p .claude

# 2. Claude Code 실행
claude

# 3. 권한 설정 요청
> .claude/settings.json 파일을 만들어줘.
  npm 명령어(test, run dev, run build, install)는 자동 허용하고,
  rm -rf, sudo 명령어는 차단해줘.
  .env 파일 편집도 차단해줘.
```

## 핵심 정리

```
권한 = Claude Code가 할 수 있는 것의 범위

기본 원칙:
✅ 처음엔 기본 모드로 시작
✅ 허용/차단 목록은 명시적으로 설정
✅ .env 파일은 반드시 보호
✅ 실수하면 Esc+Esc로 복원
✅ 큰 작업 전엔 계획 모드로 확인
```
