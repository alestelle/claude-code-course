# 섹션 25. [프로젝트3] 이벤트 관리 앱 - 시작하기

> **이 섹션을 마치면**: 이벤트 관리 앱의 기획을 완성하고 초기 프로젝트를 설정합니다.

---

## 프로젝트 개요

### 무엇을 만드는가?

```
이벤트 관리 앱 (EventHub)

기능:
- 이벤트 생성/수정/삭제
- 참가자 등록 및 관리
- 티켓 발급
- 관리자 대시보드
- 실시간 참가 현황

기술 스택:
- Next.js 14 + TypeScript
- Supabase (DB + 인증 + Realtime)
- Tailwind CSS
- Vercel 배포
```

---

## 아이디어 구체화: 린 캔버스

### 린 캔버스란?

```
린 캔버스 = 사업 계획을 1페이지로 요약하는 도구
기획자에게 친숙한 비즈니스 모델 캔버스의 간소화 버전
```

```
> /prd 명령어로 시작:

> /prd
  이벤트 관리 앱 아이디어를 린 캔버스 형식으로 작성해줘:
  - 문제: 소규모 이벤트 주최자가 참가자 관리를 위해
          엑셀과 카카오를 혼용해서 관리가 어렵다
  - 솔루션: 간단한 이벤트 생성 + 참가자 자동 관리
  - 타겟: 50명 이하 소규모 이벤트 주최자
```

## PRD와 ROADMAP 작성

### PRD 작성

```
> EventHub의 PRD를 작성해줘:

  핵심 기능 (P0 - MVP):
  - 이벤트 생성 (제목, 날짜, 장소, 정원, 설명)
  - 참가 신청 링크 생성
  - 참가자 등록 폼
  - 주최자 대시보드 (참가자 목록)

  P1 (다음 버전):
  - 이메일 자동 발송 (확인, 리마인더)
  - QR 코드 입장 체크

  P2 (추후):
  - 유료 티켓
  - 설문 기능
```

### ROADMAP 작성

```
> ROADMAP.md를 만들어줘.
  Phase 1: MVP 2주, Phase 2: 이메일 1주 로 구성해줘.
```
