# 섹션 15. [프로젝트1] 고급 - 서브에이전트

> **이 섹션을 마치면**: 복잡한 작업을 여러 에이전트로 분업해서 더 빠르고 정확하게 처리할 수 있습니다.

---

## 서브에이전트란?

```
메인 Claude Code = 팀장
서브에이전트    = 팀원

팀장이 전체 작업을 조율하고
팀원들은 각자 맡은 부분을 독립적으로 처리합니다.
```

### 왜 서브에이전트를 쓰나?

| 이유 | 설명 |
|------|------|
| **컨텍스트 격리** | 큰 코드베이스를 부분별로 처리 |
| **병렬 처리** | 여러 기능을 동시에 개발 |
| **전문화** | 각 에이전트에게 특정 역할 부여 |
| **안전성** | 실험적 작업을 격리된 환경에서 |

---

## 서브에이전트 실전 활용

### 예시 1: 프론트엔드/백엔드 동시 개발

```
메인 에이전트:
> 선수 검색 기능을 구현해줘.
  - 프론트엔드 서브에이전트: SearchBar 컴포넌트와 결과 목록
  - 백엔드 서브에이전트: 검색 API와 DB 쿼리 최적화
  두 작업을 병렬로 진행하고 완성되면 통합해줘
```

### 예시 2: 코드 분석 + 구현 분리

```
> 서브에이전트로 먼저 현재 코드베이스를 분석해줘.
  분석 결과를 바탕으로 로그인 기능 리팩토링 계획을 세워줘.
  계획이 확정되면 구현해줘.
```

### 예시 3: 리뷰 에이전트

```
> 방금 만든 검색 API를 서브에이전트로 리뷰해줘.
  보안, 성능, 코드 품질 세 가지 관점에서 각각 검토해줘.
```

```markdown

run_in_subagent: true
---

다음 기능을 구현합니다:
1. 프론트엔드 컴포넌트
2. 백엔드 API
3. 데이터베이스 스키마 업데이트

각 부분은 독립적으로 구현 후 통합합니다.
```

## 서브에이전트 vs Agent Teams

```
서브에이전트:
- 현재 세션에서 생성
- 메인 에이전트가 조율
- 완료 후 결과 반환
- 빠른 설정, 즉시 사용

Agent Teams:
- 완전히 독립된 세션
- 더 긴 작업, 더 큰 프로젝트
- 섹션 29에서 자세히 다룸
```
