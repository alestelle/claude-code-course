# 섹션 7. 슬래시 명령어와 단축키

> **이 섹션을 마치면**: Claude Code의 핵심 명령어를 자유자재로 활용할 수 있습니다.

---

## 7-1. 슬래시 명령어란?

슬래시(`/`)로 시작하는 특수 명령어입니다. 자연어 요청이 아닌 **직접 기능 실행**에 사용합니다.

```
일반 요청: "현재 모델이 뭔지 알려줘"
슬래시:    /model
```

`/` 를 입력하면 사용 가능한 명령어 목록이 자동으로 나타납니다.

---

## 7-2. 핵심 슬래시 명령어 목록

### 프로젝트 관리

| 명령어 | 기능 | 사용 시점 |
|--------|------|----------|
| `/init` | 프로젝트 초기화, CLAUDE.md 생성 | 새 프로젝트 시작 |
| `/memory` | CLAUDE.md 및 메모리 파일 확인/편집 | 프로젝트 설정 확인 |
| `/clear` | 대화 기록 초기화 | 새 주제로 전환할 때 |

### 세션 관리

| 명령어 | 기능 | 사용 시점 |
|--------|------|----------|
| `/continue` | 이전 세션 이어하기 | 작업 재개할 때 |
| `/fork-session` | 현재 대화 복사해서 다른 방향 시도 | 여러 방법 비교할 때 |
| `/compact` | 대화 압축 (컨텍스트 절약) | 대화가 길어질 때 |
| `/context` | 현재 컨텍스트 사용량 확인 | 토큰 관리 |

### 모델 및 설정

| 명령어 | 기능 | 사용 시점 |
|--------|------|----------|
| `/model` | AI 모델 변경 | 더 강력한/빠른 모델이 필요할 때 |
| `/config` | 설정 메뉴 열기 | 환경 설정 변경 |
| `/theme` | 터미널 테마 변경 | 화면 스타일 취향 설정 |
| `/vim` | vim 편집 모드 토글 | vim에 익숙한 경우 |

### 작업 도구

| 명령어 | 기능 | 사용 시점 |
|--------|------|----------|
| `/mcp` | MCP 서버 관리 | 외부 서비스 연결 |
| `/hooks` | 훅 설정 확인 | 자동화 설정 관리 |
| `/skills` | 사용 가능한 스킬 목록 | 커스텀 명령어 확인 |
| `/agents` | 서브에이전트 설정 | 에이전트 관리 |

### 특수 기능

| 명령어 | 기능 | 사용 시점 |
|--------|------|----------|
| `/btw` | 현재 맥락 유지하며 질문 | 빠른 궁금증 해소 |
| `/simplify` | 작성된 코드 품질 검토 | 코드 완성 후 |
| `/batch` | 여러 파일 병렬 작업 계획 | 대규모 변경 |
| `/feedback` | Anthropic에 피드백 전송 | 버그 리포트, 건의 |

## 7-3. 핵심 단축키

### 가장 중요한 단축키

| 단축키 | 기능 |
|--------|------|
| `Ctrl + C` | 현재 작업 취소/중단 |
| `Ctrl + D` | Claude Code 종료 |
| `Esc + Esc` | 이전 상태로 되돌리기 |
| `Shift + Tab` | 권한 모드 순환 전환 |
| `Ctrl + L` | 화면 지우기 |

### 입력 관련 단축키

| 단축키 | 기능 |
|--------|------|
| `↑` / `↓` | 이전/다음 명령 히스토리 |
| `Tab` | 자동 완성 |
| `Option + Enter` | 줄바꿈 (멀티라인 입력) |
| `Ctrl + R` | 명령 히스토리 검색 |

### 모델 관련 단축키

| 단축키 | 기능 |
|--------|------|
| `Option + P` | 모델 전환 |
| `Option + T` | 확장 사고(Extended Thinking) 토글 |
| `Option + O` | 빠른 모드(Fast Mode) 토글 |

### 작업 관련 단축키

| 단축키 | 기능 |
|--------|------|
| `Ctrl + B` | 현재 작업 백그라운드로 전환 |
| `Ctrl + T` | 작업 목록 보기/숨기기 |
| `Ctrl + O` | 상세 출력 모드 토글 |

## 7-4. 특수 입력 접두사

슬래시 외에도 특수 문자로 시작하는 입력이 있습니다:

```bash
/명령어    # 슬래시 명령어
!명령어    # 셸 명령어 직접 실행
@파일경로  # 파일 참조
```

### 실제 사용 예시

```
# 셸 명령어 직접 실행
> !ls -la
> !git status
> !npm test

# 파일 참조
> @package.json 이 파일에서 의존성 목록을 알려줘
> @src/components/Login.jsx 이 컴포넌트를 개선해줘
```

## 7-5. 자주 쓰는 조합 패턴

### 패턴 1: 긴 대화 관리
```
대화가 길어지면 → /compact → 핵심만 압축
완전히 새로 시작 → /clear
```

### 패턴 2: 모델 선택
```
복잡한 설계 작업 → /model → Opus 선택 (가장 강력)
빠른 단순 작업  → /model → Haiku 선택 (가장 빠름)
일반 작업       → Sonnet (기본값, 균형)
```

### 패턴 3: 작업 전 계획 확인
```
> /fork-session   (현재 상태 복사)
> 리팩토링 계획 보여줘 (변경 없이 계획만)
→ 좋으면 원래 세션으로 돌아가서 실행
```

## 7-6. 실습 미션

### 미션: 주요 명령어 체험해보기

```
1. /init 실행 → CLAUDE.md 생성 확인

2. /model 실행 → 현재 모델 확인

3. /context 실행 → 컨텍스트 사용량 확인

4. !git status 실행 → 직접 셸 명령 실행 체험

5. /btw 실행 → "슬래시 명령어가 몇 개나 있어?" 질문

6. /compact 실행 → 대화 압축 후 /context 다시 확인
```

## 핵심 정리

```
가장 많이 쓰는 명령어:
/init    → 프로젝트 시작
/memory  → 설정 확인
/clear   → 새로 시작
/model   → 모델 선택
/compact → 대화 정리

가장 중요한 단축키:
Ctrl+C    → 취소
Esc+Esc   → 되돌리기
Shift+Tab → 모드 전환
```
