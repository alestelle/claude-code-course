# 섹션 29. Agent Teams

> **이 섹션을 마치면**: 독립적인 AI 에이전트들이 협업하는 팀을 구성해서 대규모 작업을 처리할 수 있습니다.

---

## Agent Teams란?

```
서브에이전트 = 메인 에이전트가 생성, 완료 후 결과 반환
Agent Teams = 완전히 독립된 에이전트들이 협업

비유:
서브에이전트 = 팀장이 직접 지시하는 팀원
Agent Teams  = 자율적으로 판단하는 독립 부서들이 협력
```

---

## Agent Teams 활용 시나리오

### 시나리오 1: 대규모 리팩토링

```
팀 구성:
- 분석 에이전트: 전체 코드 분석, 개선 포인트 파악
- 프론트엔드 에이전트: UI 컴포넌트 리팩토링
- 백엔드 에이전트: API 최적화
- 테스트 에이전트: 테스트 커버리지 확인

> EventHub 전체 코드를 리팩토링해줘.
  각 영역을 에이전트 팀으로 병렬 처리해줘.
```

### 시나리오 2: 종합 코드 리뷰

```
> 이번 PR을 에이전트 팀으로 리뷰해줘:
  - 보안 에이전트: 보안 취약점
  - 성능 에이전트: 성능 이슈
  - UX 에이전트: 사용자 경험
  - 테스트 에이전트: 테스트 커버리지

  각 에이전트 결과를 종합해서 리뷰 보고서 만들어줘.
```

## Agent Teams 설정

### /agents 명령어

```
> /agents
```

서브에이전트 목록과 설정을 확인하고 관리합니다.

### 커스텀 에이전트 정의

```markdown
# .claude/agents/security-reviewer.md
---
name: security-reviewer
description: 보안 취약점을 전문적으로 리뷰합니다
model: claude-opus  # 더 강력한 모델 사용
---

당신은 보안 전문가입니다.
코드를 검토할 때 다음에 집중합니다:
- SQL 인젝션, XSS, CSRF
- API 키 노출
- 인증/인가 취약점
- 민감 데이터 처리

OWASP Top 10 기준으로 평가하고
심각도(Critical/High/Medium/Low)를 분류합니다.
```

## 실전 Agent Teams 워크플로우

```
> EventHub 프로젝트를 출시 전 점검해줘.

  에이전트 팀 구성:
  1. 보안 에이전트 → 취약점 검사
  2. 성능 에이전트 → 속도 최적화 포인트
  3. 접근성 에이전트 → WCAG 기준 체크
  4. SEO 에이전트 → 검색엔진 최적화

  각 에이전트 결과를 종합해서
  "출시 전 체크리스트"를 만들어줘.
```

---

## Agent Teams vs 서브에이전트 선택 가이드

| 기준 | 서브에이전트 | Agent Teams |
|------|------------|-------------|
| 작업 크기 | 소~중 | 대형 |
| 독립성 | 낮음 | 높음 |
| 설정 복잡도 | 낮음 | 높음 |
| 병렬 처리 | 제한적 | 완전 병렬 |
| 컨텍스트 격리 | 부분 | 완전 |
