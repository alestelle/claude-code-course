섹션 7-S에서 스킬의 기초를 배웠습니다. 여기서는 실제 프로젝트에서 활용하는 심화 패턴을 배웁니다.

---

## 스포츠 검색 서비스 전용 커맨드 세트

### /new-page: 새 페이지 생성

```markdown
# .claude/skills/new-page.md
---
description: 새 페이지를 표준 구조로 생성합니다
---

# 새 페이지 생성

인수로 받은 페이지 이름으로 다음을 생성합니다:

1. `src/app/[페이지명]/page.tsx` - 서버 컴포넌트
2. `src/app/[페이지명]/loading.tsx` - 로딩 UI
3. `src/app/[페이지명]/error.tsx` - 에러 처리

## 페이지 기본 구조
- 메타데이터(SEO) 포함
- 표준 레이아웃 적용
- TypeScript strict 모드
```

실행:
```
> /new-page player-detail
> /new-page team-profile
> /new-page match-result
```

### /add-api: API 엔드포인트 생성

```markdown
# .claude/skills/add-api.md
---
description: RESTful API 엔드포인트를 표준 구조로 생성합니다
---

# API 엔드포인트 생성

다음 구조로 API를 생성합니다:
- GET: 목록 조회 또는 단건 조회
- POST: 생성
- PUT: 수정
- DELETE: 삭제

Prisma를 사용하고, 에러 처리를 포함합니다.
응답 형식: `{ data: ..., error: null }` 또는 `{ data: null, error: "..." }`
```

### /check: 코드 품질 체크

```markdown
# .claude/skills/check.md
---
description: 변경된 파일의 품질을 종합 검토합니다
---

# 코드 품질 체크

다음을 순서대로 실행합니다:
1. TypeScript 타입 오류 확인
2. ESLint 검사
3. 미사용 import 제거
4. console.log 제거
5. 테스트 실행 (있는 경우)

문제가 있으면 자동으로 수정합니다.
```

---

## 고급 커스텀 커맨드 패턴

### 파라미터 받기

```
> /new-page player-detail --with-search --with-pagination
```

```markdown
# .claude/skills/new-page.md
---
description: 옵션을 받아 새 페이지를 생성합니다
---

사용자가 제공한 옵션을 해석합니다:
- `--with-search`: 검색 기능 포함
- `--with-pagination`: 페이지네이션 포함
- `--with-filter`: 필터 기능 포함
```

### 파일 자동 참조

```markdown
# .claude/skills/migrate.md
---
description: DB 스키마 변경에 맞춰 코드를 업데이트합니다
---

@prisma/schema.prisma 를 읽어서
스키마 변경사항을 파악하고
관련된 모든 TypeScript 타입과 API를 자동 업데이트합니다.
```

## 커맨드 목록 관리

### 팀 공유용 커맨드 문서화

```markdown
# .claude/skills/help.md
---
description: 이 프로젝트의 모든 커스텀 커맨드를 안내합니다

# 스포츠 검색 서비스 커맨드 목록

## 개발
- `/new-page [이름]` - 새 페이지 생성
- `/add-api [이름]` - API 엔드포인트 추가
- `/check` - 코드 품질 검토

## 데이터베이스
- `/migrate` - 스키마 변경 후 코드 업데이트
- `/seed` - 테스트 데이터 생성

## 배포
- `/deploy-preview` - 미리보기 배포
- `/deploy-prod` - 운영 배포
```
