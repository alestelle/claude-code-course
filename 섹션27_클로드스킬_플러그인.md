# 섹션 27. 클로드 스킬 & 플러그인 심화

> **이 섹션을 마치면**: 팀이 함께 쓰는 스킬 라이브러리를 구축하고, 플러그인 개념을 이해합니다.

---

## 스킬 심화: 팀 스킬 라이브러리

### 개인 스킬 vs 팀 스킬

```
개인 스킬 (~/.claude/skills/)
= 내 모든 프로젝트에서 쓰는 개인용

팀 스킬 (.claude/skills/ - Git에 포함)
= 팀원 모두가 공유하는 표준 명령어 세트
```

### 팀 스킬 라이브러리 구축 예시

```
.claude/skills/
├── 개발/
│   ├── new-component.md     ← 컴포넌트 표준 생성
│   ├── add-api.md           ← API 엔드포인트 표준 추가
│   └── db-migrate.md        ← DB 마이그레이션 표준화
├── 리뷰/
│   ├── review-security.md   ← 보안 관점 리뷰
│   ├── review-perf.md       ← 성능 관점 리뷰
│   └── review-ux.md         ← UX 관점 리뷰
├── 배포/
│   ├── deploy-preview.md    ← 미리보기 배포
│   └── deploy-prod.md       ← 운영 배포 체크리스트
└── 기획/
    ├── prd.md               ← PRD 작성
    ├── spec.md              ← 기능 명세서
    └── retrospective.md     ← 회고 작성
```

## 고급 스킬 패턴

### 조건부 실행 스킬

```markdown
# .claude/skills/smart-review.md
---
description: 파일 종류에 따라 적절한 리뷰를 수행합니다
---

변경된 파일 목록을 확인합니다:
- TypeScript/JavaScript → 코드 품질 + 타입 검사
- CSS/Tailwind → 스타일 일관성 + 반응형 확인
- SQL → 쿼리 성능 + 인덱스 확인
- API 파일 → 보안 + 에러 처리 확인

각 파일 유형에 맞는 리뷰를 자동으로 수행합니다.
```

### 상호작용 스킬

```markdown
# .claude/skills/feature-wizard.md
---
description: 대화형으로 기능을 구현합니다
---

다음 질문을 순서대로 합니다:
1. 어떤 기능을 만들고 싶으신가요?
2. 이 기능을 사용하는 주요 사용자는 누구인가요?
3. 성공 기준은 무엇인가요?
4. 기존 코드와 연관되는 부분이 있나요?

답변을 바탕으로 PRD → 구현 계획 → 코드 생성을 순서대로 진행합니다.
```

## 플러그인 개념

### 플러그인이란?

```
스킬 = 명령어 파일 (md)
플러그인 = 더 복잡한 기능 패키지

플러그인 포함 요소:
- 스킬 파일들
- 설정 파일
- 훅 설정
- MCP 서버 연결
- 문서
```

### 플러그인 구조 예시

```
.claude/plugins/
└── sports-service/
    ├── plugin.json          ← 플러그인 메타정보
    ├── skills/
    │   ├── new-player.md
    │   ├── update-stats.md
    │   └── generate-report.md
    ├── hooks/
    │   └── validate-data.sh
    └── README.md
```

```markdown
  "skills": ["new-player", "update-stats", "generate-report"],
  "hooks": ["validate-data"],
  "author": "홍길동"
}
```

## 실습 미션

```
> 지금까지 배운 프로젝트들을 위한
  팀 스킬 라이브러리를 구축해줘.

  포함할 스킬:
  1. /prd        → PRD 작성
  2. /review     → 코드 리뷰
  3. /deploy     → 배포 체크리스트
  4. /new-page   → 새 페이지 표준 생성
  5. /add-api    → API 표준 추가
  6. /help       → 전체 명령어 목록

  각 스킬을 .claude/skills/ 폴더에 만들고
  README도 작성해줘.
```
