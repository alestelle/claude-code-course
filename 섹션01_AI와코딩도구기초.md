# 섹션 1. AI 모델과 코딩 도구 기초

> **이 섹션을 마치면**: AI가 코드를 작성하는 원리를 이해하고, 주요 AI 코딩 도구들의 차이를 설명할 수 있습니다.

---

## 1-1. LLM이란 무엇인가

### 기획자 언어로 쉽게 이해하기

**LLM(Large Language Model)**은 '대규모 언어 모델'입니다.

쉽게 말하면, **인터넷의 거의 모든 텍스트를 학습한 뒤 "다음에 올 가장 적절한 단어"를 예측하는 시스템**입니다.

```
비유: 뛰어난 인턴 직원

- 책, 논문, 코드, 문서를 수천억 페이지 읽은 인턴
- "이런 상황에서 보통 어떻게 하더라"를 즉각 떠올림
- 단, 실제 경험이 아닌 '학습된 패턴'으로 답변
- 틀릴 수 있으므로 검토가 필요
```

### AI가 코드를 쓰는 원리

1. 사람이 자연어로 요청: *"로그인 버튼 클릭 시 팝업 뜨게 해줘"*
2. LLM이 수많은 코드 패턴에서 유사한 사례를 찾아 생성
3. 문법적으로 올바른 코드를 출력
4. Claude Code가 그 코드를 실제 파일에 적용

## 1-2. 주요 AI 모델 비교

| 모델 | 만든 곳 | 특징 |
|------|---------|------|
| **Claude** (Sonnet, Opus) | Anthropic | 코드/분석에 강함, 안전성 중시 |
| GPT-4o | OpenAI | 범용성, 가장 널리 알려짐 |
| Gemini | Google | 구글 서비스 연동에 강함 |
| Llama | Meta | 오픈소스, 자체 운영 가능 |

> Claude Code는 **Claude 모델**을 핵심으로 사용합니다.

---

## 1-3. AI 코딩 도구의 종류

### 도구 유형 3가지

```
1. 채팅형 (ChatGPT, Claude.ai)
   └─ 대화창에서 코드 요청 → 복사해서 직접 붙여넣기
   └─ 불편함: 파일에 직접 적용 안 됨

2. IDE 통합형 (GitHub Copilot, Cursor)
   └─ 코드 편집기 안에서 AI가 자동 완성
   └─ 편리하지만 '대화'가 제한적

3. 에이전트형 (Claude Code) ← 우리가 배울 것
   └─ 터미널에서 실행, 파일 직접 편집
   └─ 복잡한 작업을 자율적으로 처리
   └─ "기획서 보고 앱 만들어줘" 수준까지 가능
```

### 왜 Claude Code인가?

```
기획자 관점에서의 장점:

✅ 자연어로 지시 → AI가 알아서 구현
✅ 파일을 직접 읽고, 수정하고, 실행까지
✅ "이 기능 추가해줘" 한 마디로 여러 파일 동시 편집
✅ 테스트까지 자동 실행
✅ 기획자의 아이디어를 직접 프로토타입으로
```

---

## 1-4. AI 코딩 도구 사용 시 주의사항

### 알고 써야 하는 것들

**1) 환각(Hallucination)**
- AI가 없는 기능이나 잘못된 코드를 자신 있게 출력할 수 있음
- 해결책: 중요한 코드는 반드시 테스트 실행으로 검증

**2) 컨텍스트 한계**
- AI는 한 번에 처리할 수 있는 텍스트 양이 제한됨 (컨텍스트 창)
- 너무 긴 대화는 앞부분을 '잊음'
- 해결책: CLAUDE.md로 핵심 정보 항상 주입

**3) 보안**
- 코드에 API 키, 비밀번호 포함하면 위험
- 해결책: .env 파일 분리, 민감정보 직접 입력 금지
